The package fr.sorbonne_u.components.examples.dpp provides a data processing pipeline example to show the use of plug-ins to add new functionalities to components with a form of mixins.

The example is first programmed without plug-ins in the following packages:

  • fr.sorbonne_u.components.examples.dpp.components contains generic component definitions directly implementing the data flow among them;
  • fr.sorbonne_u.components.examples.dpp.connections contains the ports and connectors used by the generic components;
  • fr.sorbonne_u.components.examples.dpp.example contains a simple example showing how to use the generic components.
  • fr.sorbonne_u.components.examples.dpp.interfaces contains the interfaces used by the data processing pipeline.

Then, the package fr.sorbonne_u.components.examples.dpp.withPlugins and its subpackages reimplement the data processing pipeline using plug-ins. The idea is to show how orthogonal functionalities can be added to components using a form of mixins. This reimplementation is programmed in the following packages, reusing much of the above definitions:

  • fr.sorbonne_u.components.examples.dpp.withPlugins.plugins contains the three plug-ins, one for the emission of the data pieces, one for the reception of data pieces as a data sink and one for the reception of data pieces that must be processed by a host processing component;
  • fr.sorbonne_u.components.examples.dpp.withPlugins.connections contains the inbound port redefined to use the plug-ins;
  • fr.sorbonne_u.components.examples.dpp.withPlugins.example contains the same simple example but implemented with plug-ins.
